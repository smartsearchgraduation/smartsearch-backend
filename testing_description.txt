Testing ensures that our backend services are of high quality, requiring detailed planning and execution. We have decided to use the Pytest framework, as it provides extensive support for the automated testing of our Python-based architecture. Each team member also manually tests their components as they proceed; in addition to that, we have established a dedicated suite for automated tests.

We are making use of unit and integration tests in particular to cover all test cases. We have carefully planned which test cases to use and how to implement them within the Pytest framework. We test each function with specific variables to study the results generated. We also record performance statistics and success rates to understand the efficiency of our algorithms, functions, and the program in general.

These test cases are used to refine our program in order to increase accuracy to the maximum. All test case codes are uploaded to GitHub repositories, keeping track of each version with proper naming conventions. GitHub helps with following the changes made and new features added, as each update can be committed and tracked. This ensures our code versions are well-monitored and kept under control.
