# Test Plan: Backend Repository Comprehensive Tests

## Overview

This plan fills the existing test coverage gaps across all 10 sections (3.2.1‚Äì3.2.10). It adds tests organized by feature area in new test files and extends existing ones. Tests use `pytest` with `unittest.mock` for service isolation and SQLite `:memory:` for DB-backed endpoints.

## Notes

- Use `pytest` test runner with `-v` flag
- DB-backed tests use `TestingConfig` (SQLite `:memory:`) via the `create_app` fixture
- External HTTP services are mocked with `unittest.mock.patch` ‚Äî no real network calls
- File system operations (image save/delete) should use `tmp_path` fixture or `tempfile.mkdtemp()` and clean up in teardown
